#### IP address

  네트워킹이 가능한 장비를 식별하는 주소. 네트워크 상에서 통신을 하기 위해서는 통신규약(protocol)을 따라야 하는데, 규약들 중에는 "네트워킹을 하는 장비들에게 숫자 12개의 고유한 주소를 주어, 그 주소를 통해 서로를 인식하고 통신하도록 하자"라는 의미의 규약이 존재한다.

#### IPv4 주소

 IP version 4 주소. 오늘날 일반적으로 사용하는 IP 주소. 주소의 범위는 32비트로 0~255 사이의 10진수 넷을 쓰고 '.'으로 구분하여 나타냄.
 따라서 0. 0. 0. 0 부터 255. 255. 255. 255까지의 주소를 가질 수 있음. 이론적으로 42,94,967,296개의 IP가 존재하지만, 일부 번호는 특별한 용도를 위해 예약되어 있음. 이를테면, 127.0.0.1은 localhost로 자기 자신을 가리킨다.

#### IP주소의 클래스

 IP주소에는 클래스라는 개념이 있음. 클래스란, 하나의 IP주소에서 네트워크 영역과 호스트 영역을 나누는 방법이자 약속. 하나의 네트워크에서 몇개의 호스트 IP를 가질 수 있는지에 따라 클래스를 나눌 수 있다.

#### Network/Broadcast 주소

  네트워크 주소란 해당 네트워크의 첫번째 IP주소. 네트워크 주소는 IP주소와 서브넷마스크의 AND연산을 통해 구할 수 있다. 그럼 서브넷마스크란 뭘까, 서브넷마스크는 왜 사용할까..
  
  서브넷 마스크에 대해 알아보려면 다시 위의 IPv4를 떠올려야 한다. IPv4는 초기에 클래스를 통해 주소를 할당하는 방법을 택했다고 한다. 예를 들어, 어떤 기업에 B클래스를 할당했다고 하자. B클래스에서는 host address로 0.0 ~ 255.255의 범위를 사용할 수 있으니 약 65000(2^16)여개의 주소를 사용할 수 있다. 그런데 기업에서 그 중 1만개 정도의 IP만 사용한다면 나머지 약 5만개의 IP는 쓰이지 않은 채로 남는다. 그러자고 클래스 C를 할당하자니 기업의 IP자원이 너무 부족하게 된다.

  이러한 문제를 해결하기 위해 서브넷이 등장했다. 서브넷은 말 그대로 부분망이라는 뜻으로, IP주소에서 네트워크 영역을 부분적으로 나눈 부분 네트워크를 뜻한다. 이미지로 보면 아래와 같다.

  다음은 각 IP 클래스들의 기본 서브넷이다.

  예를 들어, 클래스 A의 IP가 116.81.97.8일 때, A의 서브넷 마스크는 255.0.0.0이므로 이 둘을 이진수로 변환하고 AND연산을 수행하면,
  116. 81. 97. 0->01110100.01010001.01100001.00001000
  255.  0.  0. 0->11111111.00000000.00000000.00000000
                  01110100.00000000.00000000.00000000이 나온다.
  AND연산의 결과인 116.0.0.0이 클래스 A의 네트워크 주소가 되고, 나머지부분인 ~.81.97.0이 호스트를 식별하는 호스트 ID가 된다.

  서브넷 마스크는 IP 주소 뒤에 /(숫자)와 같은 형태로 표시할 수도 있다. 이것은 서브넷 마스크의 bit수를 의미한다. 또 하나의 예를 들면, 192.168.3.19/24라는 주소가 있으면, 서브넷 마스크는 11111111.11111111.11111111.0(255.255.255.0)이 되고, 네트워크 주소는 192.168.3.0, 나머지가 호스트 ID가 된다.

  위의 예시에서 호스트에 할당할 수 있는 IP주소의 범위와 개수는 어떻게 될까??
  서브넷 마스크가 24비트였으므로 32비트 중 24비트를 뺀 나머지인 8비트만큼 경우의 수가 생긴다. 2^8은 256이지만, 0은 네트워크주소로 마지막은 브로드캐스트 주소로 할당이 되기 때문에 총 경우의 수에서 두 가지를 빼주어야 한다. 그러므로 가질 수 있는 경우의 수는 254가지, 범위는 192.168.3.1~192.168.3.254가 된다.

#### 공인 IP와 사설 IP

  IP를 구분하는 방법 중 하나는 공인과 사설의 구분이다. 공인 IP는 인터넷 사용자의 로컬 네트워크를 식별하기 위해 ISP(인터넷 서비스 공급자)가 제공하는 IP주소이다. 공인 IP는 전세계에서 유일한 IP주소를 가지며, 외부에 공개되어 있기 때문에 다른 PC들로부터 접근이 가능하다. 그러므로 방화벽 등 보안 프로그램을 설치할 필요가 있다.

  사설 IP는 일반 가정이나ㅣ 회사 내 등에 할당된 네트워크의 IP주소이며, 가상 IP라고도 한다. IPv4의 주소부족으로 인해 서브넷팅된 IP이기 때문에 라우터에 의해 로컬 네트워크상의 PC나 장치에 할당된다. 사설 IP 주소대역은 다음과 같다.
  - Class A : 10.0.0.0 ~ 10.255.255.255
  - Class B : 172.16.0.0 ~ 172.31.255.255
  - Class C : 192.168.0.0 ~ 192.168.255.255

 (참고) IP를 할당해주는 인터넷 프로토콜에는 DHCP(Dynamic Host Configuration Protocol, 동적 호스트 설정 프로토콜)가 있는데, 해당 호스트에게 IP 주소, 서브넷 마스크, 기본 게이트웨이 IP주소, DNS서버 IP주소를 자동으로 일정 시간 할당해준다.

#### TCP/IP 모델
 
 데이터는 어떻게 전송될까? 컴퓨터는 0과 1을 사용한다. 0과 1로 구성된 아주 긴 전기신호인 것이다. 그럼 이러한 전기신호를 보내면 물 흐르듯 전송되느냐 하면 그건 아니다. 한 컴퓨터 혹은 스마트폰 등에서 발생한 데이터 상대방 컴퓨터나 서버로 전송되기 위해서는 표준화된 규격(프로토콜)이 필요하다. 컴퓨터 간 데이터를 주고받을 때 에러가 발생하지 않도록 알맞게 나누어 전송하고, 수신하는 쪽에서 다시 기존의 정보로 변환하는 과정은 어떻게 약속되어 있을까.
